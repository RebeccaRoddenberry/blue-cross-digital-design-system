Right Rail

Right Rail

This sidebar is dedicated to in-context next-best-actions for the user to take.

View Specs & Styles in Zeplin open_in_new

Overview

The Right Rail aims to give users context to the page they are viewing and quick access to helpful information. Right Rail houses items not on the page to avoid redundancies. For example, if there is a component linking to a page that there’s not also a link in Related Information or Quick Links that goes to the same page.

The Right Rail should display on all top-level views. It is an optional component for secondary page levels, such as Claim Details.

Any combination of these shouldn’t take up more than 2/3 of the Right Rail height.

First release - "Help" and "Contact Us" can be found in the footer, so we will not be including it in the right rail until we have the Help Center finalized.

Content Types

(In order of their hierarchy)



1. Cards

  • Header
  • Solid background
  • CTA outlined button or link
  • No imagery
  • Single Icon acceptable (example: Nurse Line)
  • No more than one card in RR to clearly define hierarchy
  • Keep in mind that this will not always be prioroty 1 on the tablet and mobile views when we push to the bottom
  • Example: Primary Care Provider


    • 2. Information Block

  • No more than 3 to 5 bullets


    • 3. Related Information

  • No more than 5
  • No less than 3
  • Related to the page
  • FAQ link will always appear first - Single link will go to FAQ section that relates to the feature the member is viewing
  • If no related links - hide whole section


    • 4. Quick Links

  • No more than 5
  • No less than 3
  • Standard links like ID Card TBD


    • 5. RTDM Tiles

  • No more than 1 per secondary page
  • Should be relevent to the page
  • Should appear last
