Alerts

One type of notification are informational alerts that appear within the page. These display an important, succinct message in three variations:

  • Global Alert
  • Page-Level Alert
  • Component-Level Alert
Alert Session Storage Information and Content Guidelines are provided below.

Global Alerts









  • Types:
    • High alert (Red) – This could signify user error or critical information.
    • Medium priority (Yellow) - System outage, user action required, planned system outage.
    • Low priority (Blue) – Information message, supplemental info, alternative to tool tip.
  • Interaction:
    • The user can close the global alert.
    • Once closed, the alert will no longer be present during the user’s session (See alert session information)
    • The global alert is not fixed. It will scroll with the page.
  • Content:
    • Link option: The global alert may have a link or a button.
  • Placement:
    • The global alert will be present on every application page.
    • The global alert will be placed directly below the global top nav.




Page-Level Alerts











  • Types:
    • High alert (Red) – This could signify user error or critical information
    • Medium priority (Yellow) - System outage, user action required, planned system outage
    • Low priority (Blue) – Information message, supplemental info, alternative to tool tip
  • Interaction:
    • The user can close the page-level alert.
    • Once closed, the alert will no longer be present during the user’s session. See alert session information.
  • Content:
    • Link option: The page-level alert may have a link or a button.
  • Placement:
    • The page-level alert can be posted to the homepage, or any secondary page, depending on which page(s) are effected.
    • Page-level alert will be added only to page(s) effected
    • For both the homepage and the secondary pages, the page-level alert will be placed at the top of page, under the global top nav.
  • Number of alerts: 
    • The site can have both a system-level alert on the homepage and a page-level alert on the homepage. There may be multiple system and page-level alerts presented at the same time.
  • Priority: 
    • System-level alerts would take priority of placement above any page-level alerts that exist on the homepage. System-level alerts are only shown on the homepage.




Component-Level Alerts











  • Types:
    • High alert (Red) – This could signify user error or critical information
    • Medium priority (Yellow) - System outage, user action required, planned system outage
    • Low priority (Blue) – Information message, supplemental info, alternative to tool tip.
    • Success message (green)
  • Interaction:
    • The user will not be able to close the component level alerts/notifications.
    • High alert (Red) error messages will remain until the user has submitted the corrections/valid entries. Once the valid entries are submitted, the user will receive the green confirmation message. The green confirmation message will remain during the user’s session until logout.
    • If the form submission has valid entries – the green confirmation message will be shown. The green confirmation message will remain during that user’s session until logout.
  • Content:
    • The component alert may have a link or button at the end of the alert.
  • Placement:
    • The component alert/notification will be at the top of the component, underneath the component title.
    • If there are tabs within the component, the alert/notification will be placed underneath the tabs. 

Alert Session Storage Information

  • Component-level alert: 
    • User will not be able to close out of a component-level notification. 
    • The notification will not have an ‘x’ to close the notification.
  • System-level and Page-level alerts: 
    • User will be able to close out of a system-level and page-level alerts. 
    • The alert will have an ‘x’ that the user can click to close the alert. 
    • If a user closes out the alert, the alert should not appear as long as they are logged in to that session. 
    • If the user closes the alert, then closes the browser window, then opens a new window & logs back in – they will see the alert, again – as long as the alert is within the timeframe of how long it should exist on the app (provided by business). 
    • If the user closes the alert, then reloads the page, the alert should not re-appear. The alert is dismissed for as long as you are logged in to that session.
    • If the user closes the alert, they will not have a way to access the alerts while in that session.

Content Guidelines

  • Alerts should be direct and concise.
    • Start confirmation alerts with a 2-3 word statement
      • Example: Password updated.; Message sent.; etc.
    • A second sentence can add more context or instructions.
      • Example: Plan linked. Your additional plan information will appear in your current Blue Connect account.
  • Use standard hyperlinking practices within the text when necessary. Alternatively, a hyperlink with text in a call-to-action format can be placed at the end of the message.
    • Example: Your current email address is returning Blue Cross NC emails as spam. <Change Email>
  • Follow up with additional instructions where necessary, particularly error messages
    • Example: The information you entered is not correct. Please try again.
